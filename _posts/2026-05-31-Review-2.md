## 1. 논문 한 줄 요약

**QUAR-VLA는 4족보행 로봇이 카메라로 본 시각 정보와 사람의 언어 명령을 함께 이해하여, 실제 로봇이 수행할 수 있는 행동 명령을 생성하는 Vision-Language-Action 모델이다.**

기존 4족보행 로봇 연구는 주로 시각 정보만 사용하거나, 언어 명령만 사용하는 방식이 많았다. 하지만 실제 환경에서 로봇이 자연스럽게 동작하려면 주변 환경을 보고, 사람의 명령을 이해하고, 이를 행동으로 연결할 수 있어야 한다.

이 논문은 이를 위해 **QUAR-VLA**라는 새로운 문제 설정을 제안하고, 이를 학습하기 위한 **QUARD 데이터셋**과 **QUART 모델**을 제시한다.

---

## 2. 기존 방식의 한계

논문에서는 기존 4족보행 로봇의 task를 크게 두 가지로 구분한다.

## 2.1 QUAR-VA

**QUAR-VA**는 Vision-Action 방식이다.

즉, 로봇이 카메라 이미지와 같은 시각 정보를 보고 행동을 생성한다.

장점은 로봇이 주변 환경을 인식할 수 있다는 점이다.
하지만 언어 명령을 세밀하게 반영하기 어렵다.

예를 들어,

> "빨간 박스로 천천히 이동해."

또는

> "먼저 서랍으로 가고, 그다음 흰색 큐브로 가."

와 같은 복합 명령을 처리하기 어렵다.

---

## 2.2 QUAR-LA

**QUAR-LA**는 Language-Action 방식이다.

즉, 로봇이 사람의 언어 명령을 입력받아 행동을 생성한다.

장점은 다양한 명령을 표현할 수 있다는 점이다.
하지만 시각 정보가 없기 때문에 로봇이 주변 환경을 직접 인식하기 어렵다.

예를 들어,

> "왼쪽에 있는 물체로 가."

라는 명령을 수행하려면 실제로 왼쪽에 어떤 물체가 있는지 봐야 한다.
따라서 언어만으로는 자율적인 navigation에 한계가 있다.

---

## 3. QUAR-VLA의 핵심 아이디어

QUAR-VLA는 Vision, Language, Action을 하나로 연결한다.

$$
\text{Vision} + \text{Language} \rightarrow \text{Action}
$$

논문에서는 이를 다음과 같은 policy로 정의한다.

$$
\mu : S \times W \rightarrow A
$$

여기서,

| 기호    | 의미                       |
| ----- | ------------------------ |
| $S$   | 로봇이 관찰한 이미지              |
| $W$   | 자연어 명령                   |
| $A$   | 로봇의 행동 공간                |
| $\mu$ | 이미지와 언어 명령을 행동으로 매핑하는 정책 |

즉, 이미지 $s$와 언어 명령 $w$가 주어졌을 때 로봇의 행동 $a$를 출력한다.

$$
\mu(s, w) = a
$$

예를 들어,

> "Go to the red box slowly with a trotting gait."

라는 명령이 들어오면 모델은 다음을 수행해야 한다.

1. 이미지에서 red box를 찾는다.
2. slowly라는 속도 조건을 이해한다.
3. trotting gait라는 보행 방식을 반영한다.
4. 로봇이 실행 가능한 행동 명령을 생성한다.

---

## 4. 논문의 주요 기여

이 논문의 핵심 기여는 세 가지이다.

## 4.1 QUAR-VLA 문제 설정 제안

4족보행 로봇을 위한 Vision-Language-Action 문제를 제안했다.

기존처럼 시각 정보만 사용하거나 언어 명령만 사용하는 것이 아니라, 두 정보를 함께 사용하여 행동을 생성한다.

---

## 4.2 QUARD 데이터셋 구축

논문은 **QUARD**라는 대규모 4족보행 로봇 데이터셋을 구축했다.

QUARD는 다음과 같은 task를 포함한다.

| Task                            | 설명                    |          Episodes |
| ------------------------------- | --------------------- | ----------------: |
| Distinguish Letter              | 특정 글자 구분              |               10K |
| Go to Object                    | 목표 물체로 이동             | 72K sim / 3K real |
| Go through Tunnel               | 올바른 터널 통과             |               48K |
| Go to Object and Avoid Obstacle | 장애물 회피 후 목표 이동        |               63K |
| Crawl under Bar                 | 낮은 바 아래로 이동           |                1K |
| Unload Object into Receptacle   | 물체를 목표 receptacle에 넣기 |               52K |
| Total                           | 전체 데이터                |          **249K** |

QUARD는 단순 이동뿐만 아니라 장애물 회피, 터널 통과, 자세 조절, 물체 조작까지 포함한다는 점에서 의미가 있다.

---

## 4.3 QUART 모델 제안

논문은 QUAR-VLA를 해결하기 위한 모델로 **QUART**를 제안한다.

QUART는 **QUAdruped Robotic Transformer**의 약자이다.

QUART의 전체 흐름은 다음과 같다.

```text
Image + Language Instruction
        ↓
Tokenization
        ↓
Decoder-only Transformer
        ↓
Action Token
        ↓
Detokenize
        ↓
Robot Command
```

즉, 이미지와 언어 명령을 토큰으로 변환한 뒤, Transformer가 로봇 행동 토큰을 생성한다. 이후 이 토큰을 실제 로봇이 사용할 수 있는 연속적인 제어 명령으로 변환한다.

---

## 5. Action Space

이 논문에서 중요한 부분은 action space 설계이다.

4족보행 로봇은 단순히 앞으로 가는 것만으로 제어할 수 없다.
속도, 회전, 보행 패턴, 몸체 높이, 발 높이 등을 함께 제어해야 한다.

논문에서 정의한 action space는 다음과 같다.

$$
\left[
v_x, v_y, \omega_z, \theta_1, \theta_2, \theta_3, f, h_z, \phi, s_y, h_z^f, t
\right]
$$

각 요소의 의미는 다음과 같다.

| 변수                             | 의미           |
| ------------------------------ | ------------ |
| $v_x$                          | x축 방향 속도     |
| $v_y$                          | y축 방향 속도     |
| $\omega_z$                     | z축 회전 속도     |
| $\theta_1, \theta_2, \theta_3$ | gait pattern |
| $f$                            | 보행 주파수       |
| $h_z$                          | 로봇 몸체 높이     |
| $\phi$                         | pitch angle  |
| $s_y$                          | 발 너비         |
| $h_z^f$                        | 발 높이         |
| $t$                            | 행동 종료 신호     |

여기서 중요한 점은 QUART가 로봇의 관절을 직접 제어하지 않는다는 것이다.

QUART는 motor torque나 joint command를 직접 출력하지 않고, 속도나 gait 같은 **high-level command**를 생성한다. 실제 관절 제어는 low-level controller가 담당한다.

이 방식은 대형 Vision-Language Model을 로봇 제어에 적용하기 위한 현실적인 선택이다.

---

## 6. Action Discretization과 Detokenize

QUART는 연속적인 행동 값을 직접 예측하지 않는다.
대신 각 action dimension을 256개의 bin으로 나누어 discrete action token으로 변환한다.

이를 수식으로 표현하면 다음과 같다.

$$
a_d = \operatorname{Discretize}(a_c)
$$

여기서,

| 기호    | 의미                    |
| ----- | --------------------- |
| $a_c$ | continuous action     |
| $a_d$ | discrete action token |

모델은 discrete action token을 출력하고, 이를 다시 실제 로봇 제어에 사용할 수 있는 continuous action으로 변환한다.

$$
a_c = \operatorname{Detokenize}(a_d)
$$

쉽게 말하면,

```text
Action Token → Detokenize → 실제 로봇 제어 명령
```

의 과정이다.

---

## 7. QUART 수식

QUART는 이미지 $s$와 언어 명령 $w$를 입력받는다.

먼저 tokenizer $\tau$를 통해 이미지와 언어를 토큰 $t$로 변환한다.

$$
\tau(t \mid s, w)
$$

그리고 Transformer가 이 토큰을 바탕으로 action token $a_d$를 예측한다.

논문에서는 QUART를 다음과 같이 표현한다.

$$
\operatorname{QUART}(a_d \mid s, w)

===================================

p(a_d \mid t)\tau(t \mid s, w)
$$

여기서,

| 기호              | 의미                                |
| --------------- | --------------------------------- |
| $s$             | 입력 이미지                            |
| $w$             | 언어 명령                             |
| $t$             | 이미지와 언어 명령으로부터 생성된 토큰             |
| $a_d$           | 이산화된 action token                 |
| $p(a_d \mid t)$ | 입력 토큰을 바탕으로 action token을 예측하는 모델 |

즉, QUART는 이미지와 언어 명령을 토큰화하고, 이를 바탕으로 로봇 행동 토큰을 생성한다.

---

## 8. 실험 설정

논문은 QUART의 성능을 다음 두 가지 관점에서 평가한다.

1. 기존 VLM 기반 baseline보다 4족보행 task에서 더 효과적인가?
2. 학습하지 않은 물체나 새로운 언어 표현에도 일반화할 수 있는가?

평가 지표는 **Success Rate**이다.

$$
SR =
\frac{\text{Number of Successful Episodes}}
{\text{Total Number of Episodes}}
$$

비교 모델은 다음과 같다.

| 모델    | 특징                             |
| ----- | ------------------------------ |
| CLIP  | 이미지와 텍스트를 연결해서 이해              |
| R3M   | 로봇 학습에 사용할 수 있는 시각 표현 모델       |
| VC-1  | 다양한 embodied task를 위한 시각 표현 모델 |
| QUART | 논문에서 제안한 VLA 모델                |

---

## 9. 실험 결과

논문의 주요 실험 결과는 다음과 같다.

| Task          | CLIP |  R3M | VC-1 |    QUART |
| ------------- | ---: | ---: | ---: | -------: |
| Distinguish   | 0.44 | 0.58 | 0.46 | **0.66** |
| Go to         | 0.43 | 0.00 | 0.43 | **0.60** |
| Go avoid      | 0.45 | 0.00 | 0.45 | **0.53** |
| Go through    | 0.19 | 0.00 | 0.31 | **0.41** |
| Crawl         | 0.00 | 0.00 | 0.00 | **0.32** |
| Unload        | 0.00 | 0.00 | 0.00 | **0.12** |
| Unseen Object | 0.11 | 0.00 | 0.29 | **0.35** |
| Unseen Verbal | 0.14 | 0.00 | 0.19 | **0.33** |

결과를 보면 QUART가 대부분의 task에서 가장 높은 성공률을 보인다.

특히 **Crawl**과 **Unload** task가 중요하다.
이 두 task는 복잡한 움직임과 자세 제어가 필요한 hard task인데, 기존 baseline들은 모두 0%의 성공률을 보였다. 반면 QUART는 각각 32%, 12%의 성공률을 기록했다.

이는 QUART가 단순히 이미지나 언어를 이해하는 것을 넘어, 이를 로봇 행동으로 연결할 수 있음을 보여준다.

---

## 10. Generalization

논문은 QUART가 학습하지 않은 상황에서도 동작할 수 있는지 평가했다.

## 10.1 Unseen Object

학습 중 보지 못한 색상, 모양, 배경을 가진 물체에 대해 평가했다.

QUART와 VC-1은 사전학습된 VLM의 장점 덕분에 unseen object에 대해 비교적 좋은 성능을 보였다.

---

## 10.2 Unseen Verbal Instruction

학습 때 보지 못한 새로운 언어 표현에 대해서도 평가했다.

예를 들어 학습 명령이 다음과 같다면,

> Go to the object.

테스트에서는 같은 의미를 다른 표현으로 바꾼다.

> Navigate to target.

이 경우 모델은 두 문장이 같은 의미라는 것을 이해해야 한다.

실험 결과 QUART는 다른 baseline보다 새로운 언어 표현에 더 잘 대응했다.
이는 QUART가 사전학습된 multimodal large language model의 언어 이해 능력을 활용하기 때문이다.

또한 QUART는 다음과 같은 복합 명령도 어느 정도 수행할 수 있었다.

> Go to the drawer first, then go to white cube.

이는 QUART가 단순 명령뿐 아니라 순서가 있는 명령도 일부 이해할 수 있음을 보여준다.

---

## 11. 논문의 의의

이 논문의 의의는 다음과 같다.

첫째, 4족보행 로봇에 Vision-Language-Action 구조를 적용했다.

기존 VLA 연구는 주로 로봇팔 manipulation에 집중된 경우가 많았다. 하지만 4족보행 로봇은 이동하면서 균형을 유지해야 하고, gait, 속도, 몸체 자세까지 조절해야 하기 때문에 더 복잡하다.

둘째, action space를 적절한 중간 수준으로 설계했다.

QUART는 관절을 직접 제어하지 않고, 속도와 gait 같은 high-level command를 생성한다. 이 덕분에 대형 모델을 실제 로봇 제어에 적용하기 쉬워진다.

셋째, 새로운 물체와 새로운 언어 표현에 대해 일반화 가능성을 보였다.

이는 실제 환경에서 로봇이 사람의 다양한 명령을 수행하기 위해 중요한 능력이다.

---

## 12. 한계점

논문에서 제시한 한계는 다음과 같다.

## 12.1 데이터 수집 문제

대규모 로봇 데이터셋을 만드는 것은 비용과 시간이 많이 든다.
특히 실제 로봇 데이터는 사람이 직접 조작하거나 환경을 구성해야 하기 때문에 수집이 어렵다.

---

## 12.2 Sim-to-Real Gap

대부분의 데이터는 시뮬레이션에서 수집된다.
하지만 시뮬레이션과 실제 로봇 환경은 다르다.

예를 들어 센서 노이즈, 마찰력, 조명, 물체의 물리 특성 등이 달라질 수 있다.
따라서 시뮬레이션에서 잘 동작한 모델이 실제 환경에서도 항상 잘 동작한다고 보장하기 어렵다.

---

## 12.3 추론 속도

실제 로봇 제어에서는 빠른 추론 속도가 중요하다.

QUART는 high-level command를 생성하기 때문에 low-level motor control만큼 높은 주파수가 필요하지는 않지만, 실제 환경에서 더 안정적으로 동작하려면 모델 경량화와 하드웨어 가속이 필요하다.

---

## 13. 개인적인 정리

이 논문에서 가장 중요한 부분은 4족보행 로봇의 행동을 단순한 이동 명령이 아니라, 속도, 회전, gait, 몸체 높이, 발 높이, 종료 신호를 포함한 action space로 정의했다는 점이다.

4족보행 로봇은 로봇팔과 달리 이동 자체가 복잡한 제어 문제이다.
따라서 action space를 너무 단순하게 만들면 다양한 움직임을 표현하기 어렵고, 너무 세밀하게 만들면 모델이 직접 제어하기 어렵다.

이 논문은 그 중간 지점을 선택했다.

즉, QUART는 low-level joint control을 직접 수행하지 않고, low-level controller가 따라갈 수 있는 high-level command를 생성한다.

이 점이 실제 로봇에 VLA 모델을 적용하기 위한 중요한 설계라고 생각한다.

---

## 14. 결론

QUAR-VLA는 4족보행 로봇이 시각 정보와 언어 명령을 함께 이해하고, 이를 실제 행동으로 연결하기 위한 새로운 접근이다.

논문은 이를 위해 QUARD 데이터셋과 QUART 모델을 제안했다.
실험 결과 QUART는 기존 baseline보다 대부분의 task에서 더 높은 성공률을 보였고, 새로운 물체와 새로운 언어 표현에 대해서도 더 나은 일반화 성능을 보였다.

정리하면, 이 논문은 다음과 같이 요약할 수 있다.

> QUAR-VLA는 4족보행 로봇이 사람의 언어 명령과 시각 정보를 함께 이해하여 자율적으로 행동하도록 만드는 VLA 기반 접근이다.

앞으로 sim-to-real gap, 데이터 수집 자동화, 추론 속도 문제가 개선된다면 실제 환경에서 사람과 자연스럽게 상호작용하는 4족보행 로봇 연구에 중요한 기반이 될 수 있을 것이다.
