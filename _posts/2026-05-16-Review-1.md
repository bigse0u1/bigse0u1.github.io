> **논문 정보**  
> A Survey on Vision-Language-Action Models: An Action  
> Yifan Zhong, Fengshuo Bai et al. · Peking University & PKU-PsiBot Joint Lab   
> arXiv 2507.01925 · July 2025  

---

이 논문은 최근 빠르게 발전하고 있는 **VLA(Vision-Language-Action) 모델**들을 **"Action Tokenization"** 이라는 관점으로 체계적으로 분류하고 분석한 서베이 논문이다.

---

## 1. VLA(Vision-Language-Action)란?

최근 AI는 다음과 같은 흐름으로 발전해왔다.

| 구분 | 대표 모델 |
|------|-----------|
| **LFM** (Language Foundation Model) | GPT, Claude, Gemini |
| **VFM** (Vision Foundation Model) | CLIP, SAM, DINO |
| **VLM** (Vision-Language Model) | GPT-4o, Qwen2.5-VL, Gemini 2.5 Pro |

하지만 이 모델들은 대부분 **"디지털 세계"** 안에서만 동작한다.

✅ 가능한 것: 글 이해, 이미지 이해, 멀티모달 추론  
❌ 불가능한 것: **실제로 로봇을 움직이는 행동(Action)**

그래서 등장한 것이 바로 **VLA (Vision-Language-Action)** 이다.

---

## 2. VLA 기본 구조

```
Camera Image + Text Instruction
            ↓
      Vision Encoder
            ↓
        Visual Tokens
            ↓
 Language Backbone (LLM/VLM)
            ↓
       Action Tokens        ← 이것이 핵심!
            ↓
        Action Head
            ↓
      Robot Raw Action
            ↓
     Real Robot Execution
```

핵심은 단 하나.

> **"Action Head가 어떤 형태의 행동을 출력하느냐"**

논문은 이 출력 표현을 **Action Token** 이라고 정의한다.

---

## 3. 논문의 핵심 아이디어

> **"VLA 모델의 차이는 결국 어떤 Action Token을 사용하는가의 차이다."**

즉,

- 자연어로 행동할 수도 있고
- 코드로 행동할 수도 있고
- 좌표/궤적으로 행동할 수도 있고
- 이미지 목표 상태로 행동할 수도 있음

이 모든 것을 **Action Token 관점에서 통합적으로 해석**한다는 것이 이 논문의 핵심이다.

---

## 4. Action Token 8종 개요

| # | Token 유형 | 표현 방식 |
|---|-----------|----------|
| ① | **Language** | 자연어 행동 계획 |
| ② | **Code** | 코드 기반 행동 |
| ③ | **Affordance** | 상호작용 가능 위치 |
| ④ | **Trajectory** | 이동 경로 |
| ⑤ | **Goal State** | 목표 상태 이미지 |
| ⑥ | **Raw Action** | 실제 제어값 |
| ⑦ | **Latent Action** | 잠재 벡터 표현 |
| ⑧ | **Reasoning** | 행동 추론 (CoT) |

---

## 5. Action Token ① — Language Description

### 개념
행동을 **자연어**로 표현하는 방식이다

### 두 가지 수준

**High-level Plan** (의미 기반 하위 목표)
```
"찻주전자를 집어라" → "컵 위로 이동해라" → "42° 기울여라"
```

**Low-level Motion** (모터 수준 움직임)
```
"팔을 앞으로 5cm 이동" / "그리퍼 닫기"
```

### 대표 모델: SayCan

```
LLM 확률("이 행동이 지시에 적합한가?")
        ×
Value Function("이 행동을 성공할 수 있는가?")
        ↓
최종 행동 선택
```

### 장점 / 한계

| ✅ 장점 | ❌ 한계 |
|--------|--------|
| LLM 추론 능력 직접 활용 | 자연어의 근본적 모호성 |
| 해석 가능성 높음 | 정밀한 접촉 조작 어려움 |
| 인간 수정 용이 | 대형 모델 기반 → 지연 큼 |
| Long-horizon planning 강함 | — |

---

## 6. Action Token ② — Code

### 개념
행동을 **Python 코드/의사코드**로 생성한다.

```python
objects = detect_objects(image)
teapot = objects["teapot"]
robot.grasp(teapot.handle)
robot.lift(height=0.15)
robot.move_above(cup)
robot.pour(angle=42)
```

### 대표 모델 흐름
`Code as Policies` → `ProgPrompt` → `ChatGPT for Robotics` → `RoboCodeX`

### 장점 / 한계

| ✅ 장점 | ❌ 한계 |
|--------|--------|
| 복잡한 로직 구조적 표현 가능 | 사전 정의 API에 의존 |
| 조건문·반복문 활용 가능 | Symbol Grounding Problem |
| NumPy 등 외부 라이브러리 연동 | 가정과 실세계 불일치 시 실패 |
| 함수 기반 재사용 가능 | — |

> **Symbol Grounding Problem 예시**  
> 컵이 비어있다고 가정하고 코드를 생성했는데, 실제로는 물이 가득 찬 경우 → 행동 실패

---

## 7. Action Token ③ — Affordance

### 개념
물체와 **상호작용 가능한 위치**를 표현한다.

> "어디를 잡아야 하는가?"

### 4가지 하위 유형

| 유형 | 설명 | 대표 모델 |
|------|------|----------|
| **Keypoint** | 정밀한 파지 위치 (컵 손잡이 좌표) | KITE, RoboPoint, ReKep |
| **Bounding Box** | 객체 위치 영역 | GPT-4V for Robotics, A3VLM |
| **Segmentation Mask** | 픽셀 수준 윤곽 | MOO, SoFar, ROCKET-1 |
| **Affordance Map** | Heatmap 기반 상호작용 가능성 | CLIPort, VoxPoser, ManipLLM |

### 장점 / 한계

| ✅ 장점 | ❌ 한계 |
|--------|--------|
| 시각적 grounding 강함 | 대부분 2D 기반 |
| manipulation 태스크 적합 | 깊이 정보 부족 |
| 파지 위치 명확 | 시간 변화 반영 어려움 |
| — | 가려짐/모션블러에 취약 |

---

## 8. Action Token ④ — Trajectory

### 개념
시간에 따른 **이동 경로(sequence)**를 표현한다.

> "어떻게 움직일 것인가?"

### 3가지 유형

**① Point Trajectory** — K개 포인트를 시간축으로 추적
- 대표: `ATM`, `LLARVA`, `ARM4R`
- **핵심 장점**: 행동 라벨 없는 일반 비디오에서도 학습 가능 → 인터넷 비디오 활용 가능

**② Visual Trajectory** — 이미지 위에 직접 경로 시각화
- 대표: `RT-Trajectory`, `HAMSTER`

**③ Optical Flow** — 픽셀 단위 motion vector
- 장점: 전체 장면 모션 표현 가능
- 단점: 계산 비용 큼, 노이즈에 민감

---

## 9. Action Token ⑤ — Goal State

### 개념
행동 완료 후의 **미래 상태(이미지/영상)를 예측**하여 목표로 활용한다.

```
현재 이미지 (컵이 테이블 위에 있음)
        ↓
미래 상태 예측 (컵이 코스터 위에 놓인 이미지)
        ↓
그 목표를 향해 행동 생성
```

### 장점 / 한계

| ✅ 장점 | ❌ 한계 |
|--------|--------|
| 목표가 시각적으로 직관적 | 물리적으로 불가능한 이미지 생성 가능 |
| 중간 목표 생성 가능 | 생성 모델 품질에 크게 의존 |
| 복잡한 작업 분해 가능 | 추론 지연 큼 (≈10초/8프레임) |

---

## 10. Action Token ⑥ — Raw Action

### 개념
실제 로봇 **제어값을 직접 출력**한다.

```
Δx, Δy, Δz  /  Joint Position  /  Gripper Open·Close
```

### (1) Discrete Token 방식

연속 행동을 **256개 bin으로 이산화**하여 LLM 토큰으로 처리

- 대표: `RT-2`, `OpenVLA`

| ✅ 장점 | ❌ 단점 |
|--------|--------|
| LLM 구조 그대로 활용 가능 | 정밀도 제한 |
| 텍스트와 동일 토큰 공간 처리 | 고주파 제어 어려움 |

### (2) Continuous Output 방식

**Diffusion / Flow Matching** 기반 연속값 직접 출력

- 대표: `π0`

| ✅ 장점 | ❌ 단점 |
|--------|--------|
| 정밀도 높음 (최대 50Hz) | 계산 비용 큼 |
| 정교한 manipulation 가능 | LLM 토큰 구조와 불일치 |

---

## 11. Action Token ⑦ — Latent Action

### 개념
행동을 **잠재 벡터 공간**에 압축하여 표현한다.

```
행동 시퀀스 → [VQ-VAE Encoder] → z₁, z₂, z₃, ...
                                          ↓
                              VLA 모델이 잠재 토큰 예측
                                          ↓
                             [VQ-VAE Decoder] → 행동 복원
```

### 장점 / 한계

| ✅ 장점 | ❌ 한계 |
|--------|--------|
| 같은 상황에서 다양한 행동 가능성 표현 | 해석 불가능 (Black-box) |
| 압축 효율 높음 | 코드북 품질에 민감 |
| 대규모 비디오 데이터 활용 가능 | 적절한 그래뉼래리티 조절 어려움 |

---

## 12. Action Token ⑧ — Reasoning

### 개념
**"왜 그렇게 행동하는가"** 를 명시적 추론(CoT)으로 표현하는 meta-token이다.

```
추론: "컵에 물이 있으므로 수평을 유지하며 이동해야 한다."
    ↓
Language Plan: "pick up cup carefully"
    ↓
Raw Action: ΔT=[0.1, 0.0, 0.2]
```

- 대표: `ECoT`, `DriveVLM`, `AlphaDrive`, `Cosmos-Reason1`

### 장점 / 한계

| ✅ 장점 | ❌ 한계 |
|--------|--------|
| 복잡한 태스크 분해 가능 | 긴 추론 체인 → 지연 큼 |
| 해석 가능성 증가 | 실시간 고빈도 제어 부적합 |
| 디버깅·수정 용이 | — |

---

## 13. 핵심 인사이트

> **"모든 면에서 완벽한 단일 Action Token은 존재하지 않는다."**

따라서 미래의 VLA는 여러 토큰을 **계층적으로 결합**하는 방향으로 발전할 것이다.

```
Language Plan  (장기 계획, LLM 활용)
       ↓
Reasoning / Code  (논리 구조화, 추론 보강)
       ↓
Affordance  (어디를 — 공간적 그라운딩)
       ↓
Trajectory  (어떻게 — 이동 경로)
       ↓
Raw Action  (실제 제어 명령 실행)
```
