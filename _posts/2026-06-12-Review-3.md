## 1. Introduction

### Vision-Language Navigation (VLN)

VLN은 로봇이 **시각 정보 + 자연어 명령**을 이용해 목적지까지 이동하는 문제다. 그런데 기존 연구는 대부분 *시뮬레이션, 바퀴형 로봇, 고수준 이동* 중심이었다. 실제 환경에서는 계단·좁은 공간·야외 지형·장애물 때문에 훨씬 어렵다. NaVILA는 이 VLN을 **실제 다리 달린 로봇**으로 확장한다.

### Problem: Language → Low-Level Robot Actions

- 사람의 명령은 추상적이다. (예: *"방을 나가서 오른쪽으로 돌고 쓰레기통 앞에서 멈춰"*)
- 하지만 로봇은 결국 **관절 위치·속도·토크** 같은 저수준 제어가 필요하다.
- VLA가 이 저수준 명령을 *바로* 출력하면 ① 일반화가 어렵고 ② 로봇 종류가 바뀌면 다시 학습해야 한다.

### 해결책 — 2단계 프레임워크

NaVILA는 추론과 실행을 분리한다.

| 단계 | 역할 |
|---|---|
| **High-level VLA** | 이미지 + 언어 명령을 이해해 **중간 수준 행동**을 *언어로* 생성 (예: `Move forward 75cm`) |
| **Low-level Locomotion Policy** | 중간 행동을 실제 관절 움직임으로 변환, 장애물 회피·보행 제어 수행 |


### 왜 "언어 형태의 중간 수준 행동"인가? — 3가지 장점

1. **로봇 교체가 쉬움** — 저수준 실행을 VLA와 분리했으므로, 보행 정책만 바꾸면 같은 VLA를 Go2·H1·T1 등 서로 다른 로봇에 그대로 적용할 수 있다.
2. **다양한 데이터로 학습 가능** — 행동을 언어로 표현하니 사람 유튜브 영상·QA 추론 데이터까지 함께 학습할 수 있다. 덕분에 특정 명령에 과적합되지 않고 일반화·추론 능력이 강해진다.
3. **두 개의 시간 척도로 동작** — 무거운 VLA는 느리게(고수준 명령), 보행 정책은 실시간으로 돈다. 장애물 처리 같은 건 즉각 처리되어 전체적으로 안정적이다.

## 2. Method

전체 동작은 다음과 같다.

- **High-level VLA**: 현재 화면 1장 + 과거 화면들 + 언어 지시 → `Move forward 75cm` 같은 언어 명령 생성
- **Low-level Locomotion Policy**: 언어 명령을 속도 명령으로 바꾸고, LiDAR Height Map + 자기수용감각(proprioception)을 받아 **12개 다리 관절 위치를 실시간 제어**

→ VLA(저주파 추론)와 보행 정책(실시간 제어)이 협업하여 다양한 실제 환경에 일반화된다.

### 2-1. High-level VLA

**백본: VILA**
- 이미지-텍스트 기반의 효율적 VLM. 영상 인코더 대신 **이미지 기반 모델**을 쓴다(일반화·다중 이미지 추론에 강함).

**현재 vs 과거 프레임 구분**
- 현재 프레임(t) = 즉각 판단용 / 과거 프레임 = 진행상황 기억용(메모리)
- 균일 샘플링 대신, 가장 최근 프레임을 따로 떼고 과거에서 균일 추출
- `historical observations` / `current observation` 같은 언어 단서로 역할을 구분

**출력도 '언어'로 유지**
- 특수 토큰을 새로 만들지 않고 입출력을 모두 언어 영역에 둠 → 사전학습 LLM의 추론력을 최대한 활용
- 출력 행동 = `{ 왼쪽 θ도 회전 · 오른쪽 θ도 회전 · d cm 전진 · 정지 }`


### 2-2. 사람 유튜브 영상으로 학습 + SFT 데이터

데이터 파이프라인은 다음과 같다.

```
2K개 유튜브 투어영상
  → entropy 샘플링으로 궤적화
  → MASt3R로 카메라 포즈 추정해 행동 추출
  → VLM/LLM으로 지시문 생성
  → 20K 학습쌍
```

> *연속(continuous) 환경에서 사람 영상으로 내비게이션을 직접 학습시킨 첫 사례다.*

SFT 데이터는 4가지 출처를 혼합한다.

| # | 출처 | 설명 |
|---|---|---|
| ① | 실제 영상 내비게이션 | 유튜브 사람 투어영상 → 20K 궤적-지시 쌍 |
| ② | 시뮬레이션 내비게이션 | R2R-CE·RxR-CE (Habitat). 연속 동작 병합·정지 라벨 리밸런싱으로 과적합 완화 |
| ③ | 보조 내비게이션 데이터 | EnvDrop 증강 지시 + 궤적 요약 과제 + ScanQA(3D 장면 이해) |
| ④ | 일반 VQA 데이터 | 범용 시각-언어 능력 유지 → 새 장면·실세계 일반화 |


### 2-3. Low-level Locomotion Policy

- **플랫폼: Unitree Go2** — 머리 LiDAR(15Hz, 360°×90°) 장착, 다리 12개 관절만 제어(베이스 6 DoF 비구속)
- **언어 명령 → 속도 명령** — `{전진·좌회전·우회전·정지}` → 고정 속도(0.5 m/s, ±π/6 rad/s)와 시간으로 변환
- **LiDAR Height Map** — 점군 → 2.5D 높이지도. 유리·강한 햇빛에도 강건 (RGB·Depth가 실패하는 유리도 감지)
- **단일단계(single-stage) RL — PPO** — 기존 2단계 teacher-student 증류 없이 바로 학습 → 시간 효율↑, 새로운 전략 탐색 가능. Isaac Lab 레이캐스팅으로 RTX 4090에서 60K FPS 이상 고속 학습

**Sim-to-Real**: 도메인 랜덤화(질량·마찰·모터·지연)와 다양한 지형(평지·거친·경사·장애물)으로 학습 → 시뮬레이션에서 실제 로봇으로 바로 배치.

## 3. Experiments

### 평가 지표 한눈에 보기

| 지표 | 의미 | 방향 |
|---|---|---|
| **NE** (Navigation Error) | 최종 정지 위치와 목표 지점 사이 거리(m) | ↓ 낮을수록 좋음 |
| **SR** (Success Rate) | 목표 반경 안에서 성공적으로 멈춘 비율(%). **가장 핵심 지표** | ↑ 높을수록 좋음 |
| **OS** (Oracle Success) | 경로 중 한 번이라도 목표에 충분히 가까웠던 비율. SR의 상한 | ↑ 높을수록 좋음 |
| **SPL** (Success weighted by Path Length) | 성공 여부를 경로 효율(최단 대비)로 가중. 돌아가면 페널티 | ↑ 높을수록 좋음 |
| **nDTW** (normalized Dynamic Time Warping) | 정답 경로를 얼마나 충실히 따라갔는지(궤적 일치도) | ↑ 높을수록 좋음 |

### 3-1. High-level VLA ① — VLN-CE 벤치마크 (Val-Unseen)

**R2R-CE Val-Unseen · 단일 RGB 입력**

| Method | NE↓ | OS↑ | SR↑ | SPL↑ |
|---|---|---|---|---|
| NaVid (RGB) | 5.47 | 49.0 | 37.0 | 35.0 |
| *ETPNav\* (waypoint)* | *4.71* | *65.0* | *57.0* | *49.0* |
| **NaVILA (RGB)** | **5.22** | **62.5** | **54.0** | **49.0** |

> `*` = 시뮬레이터 사전학습 waypoint 사용. NaVILA는 **waypoint 미사용 모델 중 최고**.

**RxR-CE Val-Unseen · 단일 RGB**

| | NE↓ | SR↑ | SPL↑ | nDTW↑ |
|---|---|---|---|---|
| **NaVILA** | **6.77** | **49.3** | **44.0** | **58.8** |

- 고전 VLN 벤치마크 **SR +17%** 향상.
- **최초**: 단일 시점 RGB만으로 학습한 VLN 에이전트가 파노라마·뎁스·오도메트리·waypoint를 쓰는 모델과 동급 이상을 달성 → 제한된 관측을 강한 일반화가 보완한다. *추가 센서를 쓰는 기존 방법을 '적은 입력'으로 능가.*

### 3-2. High-level VLA ② — Cross-dataset & ScanQA

**Cross-dataset : R2R로만 학습 → RxR Val-Unseen Zero-shot**

| Method | NE↓ | OS↑ | SR↑ | SPL↑ |
|---|---|---|---|---|
| NaVid (단일 RGB SOTA) | 8.41 | 34.5 | 23.8 | 21.2 |
| **NaVILA** | 8.78 | **46.8** | **34.3** | **28.2** |

→ RxR 미학습 zero-shot인데도 NaVid 대비 **SR +10.5%p** (강한 교차 일반화).

**ScanQA Validation : 실제 3D 스캔 기반 질의응답 (공간 장면 이해)**

| Method | Bleu-4↑ | Rouge↑ | Cider↑ | Meteor↑ | EM↑ |
|---|---|---|---|---|---|
| NaviLLM (2D VLA) | 12.0 | 38.4 | 75.9 | 15.4 | 23.0 |
| *LEO (3D LMM, 추가입력)* | *13.2* | *49.2* | *101.4* | *20.0* | *24.5* |
| **NaVILA (8 frames)** | 14.8 | 46.4 | 95.1 | 18.7 | 27.0 |
| **NaVILA (64 frames)** | **16.9** | **49.3** | **102.7** | **20.1** | **28.6** |

→ 기존 2D VLA(NaviLLM) 대비 **CIDEr +20점 이상**. 64프레임 시 3D 스캔·RGBD·카메라 포즈가 필요한 3D 대형모델(LEO)도 *더 적은 입력으로* 능가.

### 3-3. Low-level 정책 & 신규 벤치마크 VLN-CE-Isaac

**저수준 정책 vs ROA (증류 기반)**

| | 선속도 오차↓ | 각속도 오차↓ | 충돌률↓ |
|---|---|---|---|
| ROA (w/BC) | 0.189 | 0.152 | 3.25 |
| ROA | 0.161 | 0.152 | 3.09 |
| **NaVILA** | **0.066** | **0.113** | **0.81** |

→ 증류 없이도 3개 지표 모두 우위. 특히 **충돌률 3.09 → 0.81** (장애물 회피 우수).

**VLN-CE-Isaac : Vision vs Blind**

| 설정 | NE↓ | OS↑ | SR↑ | SPL↑ |
|---|---|---|---|---|
| *Oracle* | *5.25* | *59.8* | *51.3* | *46.9* |
| Go2 Blind | 6.03 | 49.0 | 36.2 | 33.3 |
| **Go2 Vision** | **5.49** | **58.7** | **50.2** | **45.5** |
| H1 Blind | 7.67 | 33.3 | 24.4 | 21.0 |
| **H1 Vision** | **5.86** | **54.6** | **45.3** | **40.3** |

→ Vision(LiDAR) > Blind : **Go2 +14%, H1 +21%**. H1은 덩치가 커 Go2보다 난도가 높다.

**왜 새 벤치마크가 필요했나?**
- 기존 Habitat은 고수준 계획만 다루고 정밀 저수준 제어를 무시한다. (예: 소파 사이 10cm 틈도 통과 — 다리 로봇엔 비현실적)
- Isaac Sim 기반으로 관절 동작·물리 크기·접촉까지 고충실도 시뮬 → 계획~실행 전체 파이프라인을 평가. R2R 1,839개 중 통과가능 1,077개 궤적 선정.

### 3-4. Real-World 평가

- **전체 25개 지시 성공률 88%**, **복잡 지시(3+ 명령) 성공률 75%**
- GPT-4o 대비 SR·NE 모두 우위 / Go2·H1·T1 **3종 로봇에 재학습 없이 배치**

**Unitree Go2 성공률(SR) — 환경별 Simple / Complex**

| Method | WS-S | WS-C | Home-S | Home-C | Out-S | Out-C |
|---|---|---|---|---|---|---|
| GPT-4o | 0.67 | 0.33 | 0.53 | 0.00 | 0.67 | 0.50 |
| *NaVILA† (영상X)* | *0.60* | *0.73* | *0.47* | *0.40* | *0.00* | *0.00* |
| **NaVILA** | **1.00** | **0.80** | **1.00** | **0.67** | **1.00** | **0.83** |

> WS=Workspace, S=Simple, C=Complex

**사람 영상 학습 효과(ablation)**: 제거(NaVILA†) 시 Outdoor가 0%로 급락 → 추가 시 모든 환경에서 성공률 상승·오차 감소. **야외 일반화에 결정적**이다. (T1 휴머노이드도 재학습 없이 baseline 능가)

## 4. Conclusion & Limitations

### Conclusion
- NaVILA = **VLA + 보행 스킬**을 결합한 2단계 범용 내비게이션 프레임워크.
- VLA는 언어 형태의 고수준 명령 생성, 실시간 보행 정책은 장애물 회피 담당 → 추론 보존·과적합 방지·사람 영상 직접 학습으로 일반화.
- 고전 VLN +17%, 증류 기반 저수준 정책 능가, Vision > Blind, 다양한 환경·로봇에서 강한 실세계 성능.

### Limitations & Future Work

| 한계 | 내용 |
|---|---|
| 오류 회복의 한계 | 경로 이탈 시 효과적인 에러 정정을 못 해 일부 실세계 사례에서 실패 → 명시적 추론 데이터 학습으로 개선 여지 |
| 일반화·공간 이해 | 더 크고 현실적인 시뮬레이션 대규모 학습으로 다양한 시나리오·복구 사례 확보 필요 |
| 연산 비용 | 이미지 기반 VLM은 연산량이 큼(추론 약 1FPS) → 긴 컨텍스트 LLM·양자화(AWQ로 메모리 절반·속도 40%↑)로 완화 |
